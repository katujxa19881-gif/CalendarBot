# Развертывание бота на сервере (45.134.15.201) через Docker

## 1. Что будет на сервере
1. Путь приложения: `/opt/calendarbot`
2. Режим Telegram: `polling`
3. Запуск: `docker compose up -d`
4. Данные SQLite сохраняются в томе `./data` на сервере.

## 2. Подготовка сервера
```bash
apt update
apt install -y git ca-certificates curl gnupg
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
chmod a+r /etc/apt/keyrings/docker.asc
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" | \
  tee /etc/apt/sources.list.d/docker.list > /dev/null
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## 3. Клонирование проекта
```bash
mkdir -p /opt
cd /opt
git clone https://github.com/katujxa19881-gif/CalendarBot.git calendarbot
cd /opt/calendarbot
```

## 4. Настройка `.env`
```bash
cp .env.example .env
nano .env
```

Обязательно заполнить:
1. `TELEGRAM_BOT_TOKEN`
2. `ADMIN_TELEGRAM_ID`
3. `GOOGLE_CLIENT_ID`
4. `GOOGLE_CLIENT_SECRET`
5. `GOOGLE_REFRESH_TOKEN`
6. `GOOGLE_CALENDAR_ID`
7. `SENDGRID_API_KEY`
8. `SENDGRID_FROM_EMAIL`
9. `SENDGRID_FROM_NAME`

И выставить:
1. `TELEGRAM_TRANSPORT=polling`
2. `HOST=0.0.0.0`
3. `PORT=3000`

## 5. Запуск
```bash
cd /opt/calendarbot
docker compose up -d --build
```

## 6. Проверка
```bash
docker compose ps
docker compose logs -f --tail=200
```

Ищем в логах:
1. `app_started`
2. `telegram_polling_started`
3. отсутствие `startup_error`

## 7. Обновление версии
```bash
cd /opt/calendarbot
git pull
docker compose up -d --build
```

## 8. Остановка
```bash
cd /opt/calendarbot
docker compose down
```
