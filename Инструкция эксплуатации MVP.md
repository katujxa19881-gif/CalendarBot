# Инструкция эксплуатации MVP

## 1. Запуск
1. Установить зависимости: `npm install`
2. Проверить `.env` (Telegram, Google, SendGrid, админ ID).
3. Запустить бота: `npm run dev`
4. Проверить health: `GET http://127.0.0.1:3000/health`

## 2. Базовые команды в Telegram
1. `/start` — главное меню.
2. `/history` — последние заявки.
3. `/my` — последние заявки (быстрый доступ).
4. `/version` — версия бота и PID процесса.

## 3. Что делать администратору
1. Входящие заявки обрабатывать кнопками `Подтвердить` / `Отклонить`.
2. Для отклонения можно отправить комментарий (или `-` без комментария).

## 4. Что делать пользователю
1. В `Мои заявки (4)` доступны действия:
   - `Перенести` (для подтвержденной/перенесенной встречи),
   - `Отменить`.
2. После переноса/отмены проверить итоговое уведомление в Telegram.

## 5. Проверка готовности перед запуском
1. Выполнить: `npm run stage8:verify`
2. Должно вернуться `ok: true`.

## 6. Операционные проверки
1. Логи приложения: `stdout` процесса (`npm run dev`).
2. Ключевые события в логах:
   - `meeting_request_submitted`
   - `approval_confirmed` / `approval_rejected`
   - `reschedule_requested` / `reschedule_completed`
   - `cancellation_completed`
   - `email_sent` / `email_failed`
   - `retry_scheduled`
   - `integration_error`

## 7. Если бот не отвечает
1. Проверить, что запущен только один процесс бота.
2. Проверить `TELEGRAM_TRANSPORT` в `.env`:
   - локально: `polling`,
   - production: предпочтительно `webhook` (с `TELEGRAM_WEBHOOK_SECRET`).
3. Перезапустить процесс бота.
